# 依赖接口支持

使vDSO得以依赖一些自身定义、由vDSO外部代码实现的接口。类似于`crate_interface`。

接口类型：纯函数；trait（类型及其关联函数）。前者为后者的特殊情况。因此只需考虑后者。

vDSO中的定义：

```Rust
pub trait TestIf {
    fn test_fn1(&self, arg: usize) -> usize;
    fn test_fn2(&mut self, arg: usize) -> usize;
    fn test_fn3(arg: usize) -> usize;
}
```

外部代码的实现：

```Rust
pub struct TestImpl{...}

impl TestIf for TestImpl{...}
```

类型通过指针传递，关联函数通过虚函数表实现。虚函数表实现为私有数据，以支持不同地址空间中的不同实现。

## 封装vDSO内部的虚函数表调用

在vDSO中定义一个`VirtImpl`类型，实现对应的`trait`，实现方式是从函数表中调用。

`VirtImpl`类型的引用可以和指针互相转化。

这样，在vDSO内部调用时可以直接调用`VirtImpl`的方法，从而封装了各种指针转化、查询虚函数表的过程。

## 虚函数表的初始化

每个虚函数表生成一个初始化函数：

```Rust
pub static FN_TABLE: LazyInit<[*const (); 3]>;

fn init_table<T: TestIf>() {
    FN_TABLE.init_by([
        T::test_fn1 as _,
        T::test_fn2 as _,
        T::test_fn3 as _,
        ]);
}
```

## 如何用宏实现

在vdso_helper中，对`trait`声明一个宏，其需要生成以下几项内容：

|项目|需要动态生成的内容|
|-|-|
|虚函数表|变量名、数组的长度|
|虚函数表初始化函数|函数名、泛型参数、实现|
|`VirtImpl`类型|类型名|
|`VirtImpl`类型的实现|调用虚函数表的函数的实现；与指针相互转化的函数的实现？|

作为示例，宏需要把`TestIf`的定义展开成如下的内容：

```Rust
pub trait TestIf {
    fn test_fn1(&self, arg: usize) -> usize;
    fn test_fn2(&mut self, arg: usize) -> usize;
    fn test_fn3(arg: usize) -> usize;
}

pub(crate) static TestIf_TABLE: lazyinit::LazyInit<[usize; 3]> = lazyinit::LazyInit::new();

/// 需要在api.rs中创建一个同名的api
pub(crate) fn init_vtable_TestIf(test_fn1: usize, test_fn2: usize, test_fn3: usize) {
    TestIf_TABLE.init_once([test_fn1, test_fn2, test_fn3]);
}

pub(crate) struct TestIfVirtImpl;

impl TestIfVirtImpl {
    pub unsafe fn from_ptr(ptr: *const ()) -> &'static Self {
        unsafe { &*(ptr as *const Self) }
    }

    pub unsafe fn from_mut(ptr: *mut ()) -> &'static mut Self {
        unsafe { &mut *(ptr as *mut Self) }
    }

    pub fn to_ptr(&'static self) -> *const () {
        self as *const Self as _
    }

    pub fn to_mut(&'static mut self) -> *mut () {
        self as *mut Self as _
    }
}

impl TestIf for TestIfVirtImpl {
    fn test_fn1(&self, arg: usize) -> usize {
        let f: fn(&Self, usize) -> usize = unsafe { core::mem::transmute(TestIf_TABLE[0]) };
        f(self, arg)
    }

    fn test_fn2(&mut self, arg: usize) -> usize {
        let f: fn(&mut Self, usize) -> usize = unsafe { core::mem::transmute(TestIf_TABLE[1]) };
        f(self, arg)
    }

    fn test_fn3(arg: usize) -> usize {
        let f: fn(usize) -> usize = unsafe { core::mem::transmute(TestIf_TABLE[2]) };
        f(arg)
    }
}
```

难点在于函数参数的“`&self`, `&mut self`, 没有`self`”三种情况。其余应该用过程宏就可以实现。
